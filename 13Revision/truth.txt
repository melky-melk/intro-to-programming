we can see that this is sort of similar to the a or b statement, but just flipped around.

if we negate the earlier statement of ((not a) and (not b)). then we are able to create a version of the or statement using only "not" and "and" statements. So instead of False and False giving true, False and False gives False. 
Instead of every combination with True giving False, every combination with True gives True
